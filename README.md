## 🔀 브랜치 전략 개요
main ← 최종 제출 / 배포용 (팀장 관리)
dev ← 통합 개발 브랜치 (팀원들 업스트림)
├─ 팀원명1 : 본인 로컬(노트북)에서 개발 후 변경사항을 푸시할 브랜치. 푸시 후 dev로 풀 리퀘스트 후 오류가 없다면 브랜치 제거.
├─ 팀원명2
├─ 팀원명3
└─ 팀원명4

## 1️⃣ 브랜치 역할

### 🔹 main
- **최종 결과물 브랜치**
- 제출 / 시연용
- ❗ **팀장만 관리**
- 팀원은 직접 작업 ❌

---

### 🔹 dev
- 모든 기능이 모이는 **통합 개발 브랜치**
- 팀원 작업의 기준 브랜치
- Pull Request의 **병합 대상**

---

### 🔹 개인 브랜치 (팀원 이름)
- 각 팀원의 작업 공간
- dev 브랜치 기준으로 생성
- 작업 완료 후 dev로 PR 요청
